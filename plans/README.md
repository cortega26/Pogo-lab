## Deployment-readiness review findings (human-owned — NOT executor plans)

These live in the WIP `infra/` scaffold (PR-21). They are flagged separately per
the audit's guidance: they are not cleanly gate-verifiable and are owned by the
human doing the OCI deploy. Action them before beta. (Context: the OCI hosting is
actively in flux — A1 capacity fallback to a micro 1 GB instance — so some of
these may already be moving.)

- **F12 — No TLS in the deploy.** `infra/nginx/default.conf` listens on `:80`
  only; `compose.prod.yaml:44` maps `:443` with **no** 443 server block and no
  cert provisioning. As-is, auth/sessions/passwords transit in cleartext (which
  is why `SECURE_SSL_REDIRECT` was made env-optional in `2db68e3`). Add a TLS
  terminator (Let's Encrypt/certbot or a reverse proxy) + a 443 server block
  before going live, then set `SECURE_SSL_REDIRECT=True`.
- **F13 — nginx won't start as written.** `infra/nginx/default.conf:20` uses
  `${ALLOWED_HOST}`, but the file is mounted **directly** to `conf.d/`
  (`compose.prod.yaml:40`), not as an `envsubst` template, so nginx errors with
  "unknown variable" at boot. Either use the `nginx` image's
  `/etc/nginx/templates/*.template` + `envsubst` mechanism, or hardcode/`$host`.
- **F14 — Weak default secrets.** `compose.prod.yaml:6,25` default the DB
  password to `change-me-in-production`; `config/settings/base.py:21` falls back
  to the insecure default `SECRET_KEY` in prod instead of failing fast (`check
  --deploy` only *warns*, W009, and CI runs it under **test** settings). Make
  prod require both (raise if unset) and set real values in `.env.prod`.
- **F15 — `Deploy` workflow runs on every push before OCI exists.**
  `.github/workflows/deploy.yml` triggers on `push: [main]`; it fails on every
  commit today (confirmed red). Gate it to `workflow_dispatch` (or an environment
  with required secrets) until the VM/secrets are provisioned.
- **F16 — Logging hardening (low).** `apps/core/logging_filters.py` does no PII
  redaction, and `apps/core/middleware.py:10-16` reflects an unvalidated,
  client-supplied `X-Correlation-Id` into logs. Low impact today (no app code
  logs PII). If PII logging is ever added, add a real scrubbing filter; bound/
  validate the correlation-id (`[A-Za-z0-9-]`, length cap).

Not deeply audited (human-owned, WIP): the OCI provisioning scripts
`bin/{setup,assist,test}-oci.sh` (only `backup.sh`/`restore.sh` were read — both
clean, `set -euo pipefail`, confirm-before-overwrite).

## Findings considered and verified clean / rejected (do not re-audit)

- **Secrets in git history**: clean. Only `.env.example` was ever committed; the
  private keys under `infra/keys/` and all `.env*` were never tracked. **No
  rotation needed** (see Plan 002 for the image-leak prevention).
- **IDOR / object-level authz on web views**: clean. The only URL→object lookup
  is `apps/trades/views.py:55` (`get_object_or_404(..., owner=request.user)`);
  every service query is owner-scoped.
- **Missing auth / CSRF on POST / SQL injection / mass assignment**: clean (no
  `@csrf_exempt`, no `.raw()/.extra()/eval`, no `Model(**request.POST)`).
- **Private `notes` in aggregates/public/dataset**: clean — excluded from the
  community dataset, CSV export, and the account self-export observations.
- **Account export/delete privacy**: strong. Export excludes `notes` and is
  own-data-only; delete is owner-scoped and anonymizes the email;
  `AuditEvent.log` stores **no** PII (M7-1 holds). Well tested (adversarial
  sentinel tests).
- **Engine purity**: enforced by the import-linter `engine-purity` contract; no
  `engine/` module imports Django.
- **Calculator "unbounded n = DoS"**: **downgraded/rejected** — the engine math
  is closed-form (`p_at_least_one`, `p_zero`), so large `n` is fast. Plan 004
  keeps the input validation (and a sane `n` cap) but there is no compute-DoS.
- **Language-switcher "open redirect"**: **rejected** — `next` is built from
  server-side `request.get_full_path` and Django's `set_language` validates via
  `url_has_allowed_host_and_scheme`; not attacker-controllable into `//evil`.
  (The real bug was the *broken* switch — Plan 001.)
- **SEC-05 unscoped moderation services** (`apps/audit/services.py:18,40`,
  `.get(pk=)` without owner scope): **latent only** — the `audit` app ships no
  urls/views and isn't in `config/urls.py`, so not web-reachable. Left as a note
  for whoever wires a moderation UI (require a moderator permission + scope then).
- **SEC-06 `translation.body|safe`** (`content/page.html:23`): by-design trusted
  CMS pattern (staff-only admin authoring + CSP nonce). Not a finding; sanitize
  on save only if untrusted authoring is ever added.
- **PERF-02 redundant COUNTs / PERF-04 dead `select_related`**: real but minor;
  folded conceptually into 006/008's neighborhood, not separately planned.
